## 📖 Overview

The **8085 Microprocessor Simulator** is an open-source educational web application that helps students learn **8085 Assembly Language Programming** through an interactive virtual laboratory.

Instead of being just another emulator, this project focuses on **learning by visualization and practice**, allowing users to understand how instructions affect registers, memory, and processor execution in real time.

Whether you're preparing for university practicals, learning microprocessors for the first time, or revising 8085 concepts, this simulator aims to provide a clean and intuitive learning experience.

# 🎯 Planned Features

## 💻 Interactive Playground

A clean virtual workspace where users can:

- Build Assembly programs
- Execute instructions step-by-step
- Observe processor behavior
- Learn through experimentation

---

## 📝 Live Instruction Workspace

Unlike traditional simulators that use a normal code editor, this project introduces a **live instruction workspace** inspired by a laboratory notebook.

Instructions appear line-by-line as they are added, making the experience feel similar to writing programs during college practical sessions.

Example:

001 MVI A,05H

002 MVI B,03H

003 ADD B

004 HLT

This approach keeps the interface clean while making Assembly programming more approachable for beginners.

---

## 🧪 Guided Practice Mode

One of the key features of this simulator.

Students can complete practical experiments through guided step-by-step instructions.

Example:

✔ Reset the simulator

✔ Load Register A

✔ Load Register B

✔ Execute ADD instruction

✔ Observe Register A

✔ Program Completed

This feature is designed especially for laboratory sessions and self-learning.

## 💾 Browser Auto Save

Inspired by applications like Excalidraw.

Programs will automatically be saved inside the browser, allowing users to continue exactly where they left off without manually saving their work.

Features planned:

- Auto Save
- Restore Previous Session
- Recent Programs
- Project Naming

Initially, this will use browser storage (Local Storage), with cloud synchronization planned for future releases.
